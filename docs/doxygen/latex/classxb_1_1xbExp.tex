\doxysubsection{Detailed Description}
Class for handling expressions. 

The \mbox{\hyperlink{classxb_1_1xbExp}{xb\+Exp}} class is used for parsing and evaluating expression.

The Xbase64 library includes an expression parsing module which assists application programmers by providing a high level data manipulation tool and also allows for building complex index keys.

The functions included were derived from d\+BASE III Plus, Dbase IV and Clipper.~\newline
~\newline


\doxyparagraph*{Internal fuctioning}

The expression module works in two phases. Firstly, method {\itshape Parse\+Expression} is called and builds an expression tree from all the components of the expression. The tree is made up of individual nodes. The expression is checked for valid field names, literals, operands and functions. Any field references are resolved. If fields are used in an expression and the database name for the field is not included in the name with the -\/\texorpdfstring{$>$}{>} operand, the routines assume the associated database has been successfully opened. ~\newline
 Secondly, method {\itshape Process\+Expression} is called to process the expression tree created by \mbox{\hyperlink{classxb_1_1xbExp_a2c3212c64d9281f58200c63c05e93334}{Parse\+Expression()}}. The routine parses each node in the expression tree, executing functions, processing operands and manipulating data to produce the desired result.~\newline


If an expression will be processed repeatedly, it is best to pre-\/parse the tree using {\itshape Parse\+Expression}, then for each new call to the expression, execute method {\itshape Process\+Expression} which processes the tree.~\newline
~\newline


\doxyparagraph*{Expression Return Types}

Expressions will return a type of CHAR, NUMERIC, DATE or LOGICAL.~\newline


An expression return type can be determined with method {\itshape  Get\+Expression\+Result\+Type} after parsing it.~\newline


Expressions returning a return type of CHAR are limited to a 200 byte internal buffer. There is also a 100 byte limit for NDX index key support. If the 200 byte limit is not large enough for your application, adjust field {\itshape enum \{ Work\+Buf\+Max\+Len = 200 \};} in file {\itshape exp.\+h}.~\newline
~\newline


\doxyparagraph*{Expression Functions}

Each expression function also has a corresponding C++ function. It is slightly more efficient to call the C++ functions directly, rather than execute the expression parsing routines.~\newline
~\newline


\doxyparagraph*{Expression Components}

Expressions are made up of one or more tokens. A token is one of literal, database field, operand or function. Literals are either numeric or character. Character literals are enclosed in \textquotesingle{}single\textquotesingle{} or \char`\"{}double\char`\"{} quotes. numeric literals are a series of one or more contiguous numerals, \char`\"{}.\char`\"{}, \char`\"{}+\char`\"{} or \char`\"{}-\/\textquotesingle{}\char`\"{}. ~\newline
~\newline
 A field is simply a field name in the default database, or is in the form of database-\/\texorpdfstring{$>$}{>}fieldname.

See program /source/examples/xb\+\_\+ex\+\_\+expression.cpp for examples of how to use this functionality.
